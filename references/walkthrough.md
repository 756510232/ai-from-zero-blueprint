# 实战演练：从零做一个「张雪峰式高考志愿网页应用」

> **方向声明（重要）**：本演练是**用方法论从零做一个新项目**，不是"复刻某个已有系统"。
> 下面引用的 `D:\各大院校录取数据\...` 代码只是"优秀范式样例"——用来对照"我的内核落点对不对"，
> 而非照抄。真实流程是：brainstorming 确认终点 → A 主动侦察 → B 拆解（四要素）→ C 建造（真做出来）。
>
> 这正是纠正旧版"复盘=复现"错误的样本：终点是用户的产品意图，不是元任务"复盘某系统"。

---

## 第零步：brainstorming 确认「你到底想要什么」

**用户原始意图**："用 AI 给高考生和家长做志愿建议，要带张雪峰式的直白表达。"

AI 加载 superpowers `brainstorming`，一次一问（多选优先），逐步澄清终点：

**Q1（产品形态）**：做成网页应用 / 微信小程序 / CLI？ → 用户选 **网页应用**（家长考生都能开）。
**Q2（受众与痛点）**：主要给谁用？ → "高考生 + 家长，痛点是不懂'我这分数能冲哪些、稳哪些、保哪些'，怕滑档。"
**Q3（成功标准，可衡量）**：做到什么算成功？ → "输入分数/位次/选科，输出**冲/稳/保三档院校清单 + 每档一句直白理由**；理由里不编造不存在的分数线。"
**Q4（硬约束）**：数据从哪来？表达风格？ → "用公开历年录取数据；表达要张雪峰式（直白、戳痛点、说人话）；**绝不编造分数/位次**。"
**Q5（明确不做什么 / YAGNI）**： → "不替考生填志愿、不把'录取概率'当绝对结论、不做付费咨询、不接第三方广告。"

**终点锁定**（写入 `docs/plans/2026-07-15-gaokao-advisor-design.md`）：
> 一个网页应用，输入考生分数/位次/选科/省份，基于公开历年录取数据，输出冲/稳/保三档院校清单与张雪峰式直白理由；内核判定确定性、AI 只翻译；缺数据显式降级。

> ✅ 注意：这里 AI **没有**把"复盘高考志愿系统"当目标——那是元任务，错了。终点是产品。

---

## A 侦察（AI 主动套模板，不反问用户已知信息）

AI 套 `blueprint-template.md`，主动产出：

**① 概念表**（节选）
| 术语 | 人话类比 | 系统角色 |
|---|---|---|
| 位次(rank) | 全省考生排队你排第几 | 冲稳保判定的核心输入 |
| 批次线 | 一本/二本及格线 | 路由与判定的基准 |
| 冲/稳/保 | 摸高/稳妥/保底三档 | 内核输出的三分类 |
| 滑档 | 填的全没录上 | 要规避的风险，理由里提醒 |

**② 三层落点**（套模板 §1）
- ① **确定性内核**：冲稳保判定 = 拿考生位次/分数 vs 院校历年 `min_score/min_rank` 的 SQL 查库判定（**零 LLM**）；检索路由 = 问题分类后查对应知识库。
- ② **调度/路由层**：判断用户问的是"院校/专业/政策/分数趋势/家庭情况"哪类，转给对应库。
- ③ **AI 解释层**：只消费内核判定的三档清单 + 命中知识库片段，生成张雪峰式回答。**不参与冲稳保判定本身**。
- 铁律自检：判定文件 `grep` 不到 LLM import → 后续建造必须守住。

**③ 陷阱**（套模板 §2 的 6 类，给本项目实例）
- T1 口径：同校不同年分数线波动；"位次"比"分数"更稳（分数通胀）。
- T3 泄漏：不能用当年未发生的录取结果训练/判定。
- T4 编造：LLM 不能现编"某校去年 620"——必须来自库。
- T5 降级：库缺某校数据 → 标"无数据"，不编。
- T6 合规：不承诺录取、不替代官方填报系统。

**④ 数据清单**（套模板 §3）
| 数据 | 形态 | 可得性 | 降级 |
|---|---|---|---|
| 历年录取分数/位次 | 结构化(DB) | 公开可得 | 缺年份→标注"无数据" |
| 院校/专业库 | 结构化+文档 | 公开可得 | 解析失败→跳过该项 |

**⑤ PROVE**：冲稳保用"位次差"判定是高考咨询成熟做法（张雪峰本人即按位次讲）；数据用各省考试院公开数据。

## B 拆解（套标准 6 phase + 每 phase 四要素）

AI 套模板 §4，给出下表（用户重点看"边界与验收口径"）：

| Phase | 约束目标 | 完成标准 | 风险 | 检测方法 |
|---|---|---|---|---|
| **P1 Schema** | 定义考生输入表 + 院校录取表字段 | 表结构文档 + 可建表 | 字段漏口径 | 对照 T1 检查含 rank/score/年份 |
| **P2 最小内核** | 用假数据跑通冲稳保判定，**不接 AI/不接库** | 给假位次，输出冲/稳/保三档 | 位次差符号写反 | 手算 1 样本对照 |
| **P3 接真实数据** | 接 `gaokao_2025.db`，处理缺失 | 真实样本出三档 | 跨源别名/年份错配 | 抽 5 校对已知答案 |
| **P4 接解释层** | LLM 只消费三档+知识库生成回答 | 输出张雪峰式文案 | LLM 现编分数 | 回答里数字均能在命中片段找到 |
| **P5 健壮性降级** | 库缺数据/LLM 失败有 fallback | 缺数据标"无数据"；LLM 失败回退模板 | 外部漂移/超时 | 断网模拟 |
| **P6 验证声明** | 对真实考生样本验收 + 诚实声明 | 验收通过 + 声明降级项 | 过拟合错觉 | 多样本复核 |

**总步数 = 6**（固定模板实例化，不用临时发明）。高风险：P2（符号）、P4（编造）。

---

## C 建造（逐 phase，照四要素检测）

**P2 最小内核（先不接 AI/库，假数据跑通）**
AI 写出判定函数（参考样例落点：`server.js` 的 `inferRankBand()`）：
```js
// 内核：位次差 = 用户位次 - 院校历史位次（负数=用户占优→可冲）
function inferRankBand(userRank, histRank) {
  const diff = userRank - histRank;        // 负=占优
  if (diff <= -histRank*0.05) return '冲';
  if (diff >=  histRank*0.10) return '保';
  return '稳';
}
```
**验收（检测方法）**：手算样本——用户位次 7000、某校历史 12000 → diff=-5000（占优）→ '冲'。对。
> ⚠️ 这里正是 Skill"验证点显性"要抓的：若写成 `diff = histRank - userRank` 符号反了，样本会错判成'保'。建造阶段必须手算复核，这就是四要素里"检测方法"的价值。

**P3 接真实数据**：把假数换成 `gaokao_2025.db` 查询（`buildScoreInference()` 读 `min_score/min_rank`），缺年份标"无数据"。
**验收**：抽 5 校对考试院公开答案，一致。

**P4 接解释层**：LLM 消费三档 + 命中知识库片段，按 systemPrompt（张雪峰式、禁编数字）生成。参考落点：`server.js` 的 `callOpenAICompatible()` + 路由 `route-question.js` 的 `classify()`。
**验收**：回答里每个数字都能在命中片段找到 → 未编造。

**P5 健壮性**：DB 不可用 → 降级为"按知识库文档分析"并标注；LLM 失败 → 回退模板文案。
**验收**：断网模拟，不 500。

**P6 验证声明**：多样本复核通过；声明"某省 2025 数据未全→部分标无数据；录取概率不作绝对结论"。

## 诚实复盘（收尾，用方法论自检）

按本 Skill 反模式红旗 + 四要素回头看：
- ✅ 内核（冲稳保 SQL 判定）零 LLM import → 过铁律。
- ✅ 数字全溯源到 `gaokao_2025.db` → 过铁律。
- ⚠️ 真实样例项目暴露的可改进点（用方法论反向揪出，非复刻）：
  1. 外部依赖偏厚（Node + Qdrant + LM Studio + API Key 四件套）——应标"最小可运行子集"，先让 P1–P3 纯本地跑通。
  2. 位次差符号约定需在 P2 就写单测锁死（见上 ⚠️），避免 P3 接真数据后才发现反了。
  3. 数据版本需标注（"基于 2025 公开数据"），避免被当成绝对结论。

**结论**：用这套方法，从"做一个张雪峰式志愿网页应用"这个产品意图出发，无需懂高考领域，
也能逐步做出稳健、可验证、可溯源、拔掉 LLM 仍成立的内核。参考仓库只是样板和避坑镜。

---

## 这套演练印证的方法论要点
1. **终点来自 brainstorming，不是元任务**——"复盘/复现"是错的，"做产品"才对。
2. **通用部分已固化**——三层/陷阱/数据/phase 顺序 AI 主动套，不啰嗦问用户。
3. **拆解价值在四要素**——用户看懂每步边界与检测法，建造才能逐项验。
4. **从零做出 ≠ 复刻**——参考仓库是范式样例与避坑镜，不是要抄的对象。
