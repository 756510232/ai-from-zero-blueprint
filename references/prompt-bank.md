# 提问模板库（A 侦察 / B 拆解 / C 建造）

> 直接复制给 AI（或让用户照着问 AI）。目标：在**你也不懂领域**时，逼 AI 当领域分析师 + 架构师，并把说法 PROVE 出来。所有模板中的 `[...]` 替换成你的具体需求。

---

## A 侦察：逼 AI 当领域分析师

**A1 · 概念对齐**
```
我完全不懂[领域，例如"世界杯夺冠概率预测"]。请你扮演该领域的资深分析师 + 系统架构师。
先给我一张"概念表"：列出做[具体目标]必须懂的 8-12 个专业术语，每个配一句外行能懂的类比，
以及它在系统里的角色。任何说法都要给出"为什么可信"（文献 / 成熟做法 / 数据出处）。不要写代码。
```

**A2 · 分层架构（强制三层）**
```
基于上面的概念，画出这个系统的分层架构。严格按三层：
① 确定性 / 检索内核（真正产生数字 / 排名 / 抽取的部分）
② 调度 / 路由层（决定调哪个内核、何时调、怎么组合）
③ AI 解释层（只翻译，不计算）
每层列出它负责什么、明确不负责什么。如果某层当前不需要 AI，直接写"无 AI"。
```

**A3 · 坑点 + 数据源**
```
这个领域最常见的 5 个数据 / 口径 / 合规陷阱是什么？
做这个系统需要哪些数据？哪些免费可得、哪些付费、哪些根本拿不到？
拿不到的怎么降级（代理值 / 占位 / 跳过）？降级后对外怎么标注才诚实？
```

## B 拆解：逼 AI 列有序 phase

**B1 · 有序 phase + Gate**
```
把我们刚才的架构切成有序 phase。每个 phase 写：
- 子目标
- 硬性约束
- 完成标准（必须可客观检查，例如"跑通 X 条样本、数值落在 Y 区间、测试通过"）
标注哪个 phase 高风险（易过拟合 / 易泄漏 / 数据难拿），并给分支：某 phase 失败时的备选方案。
最后明确告诉我：做这个一共几步（总步数）。
```

**B2 · 已知终点 → 倒推 phase 闸门**
```
我们已经知道最终要交付[终点描述，例如"能输出 32 强夺冠概率 + 72 场预测的报告"]。
把它倒推成可控 phase，每个 phase 结束后有一个 Gate（客观验收标准）。
一次只描述一个 phase，我验收通过再进下一个。不要一次性甩整套架构。
```

## C 建造：逐 phase 交付 + 收尾诚实声明

**C1 · 单 phase 启动**
```
现在只做 phase [N]：[子目标]。
约束：[硬性约束，例如"禁止用未来数据""缺值必须标注不允许编造"]。
完成标准：[可客观检查的标准]。
做完用[验证方式，例如"对 10 条真实样本跑一遍看数值是否合理"]验收，不过关就回滚。
不要顺手做后面的 phase。
```

**C2 · 收尾诚实声明（必做）**
```
全部 phase 完成后，列一张表：
- 哪些输出已验证（怎么验证的）
- 哪些只是推测 / 未验证
- 哪些用了降级 / 代理值（标在原输出什么位置）
- 复现需要什么（随机种子 / 数据版本 / 依赖版本）
不要省略任何一项。
```

**C3 · 反模式自检**
```
回顾我们做的整个系统，逐项检查这些红旗，有就修：
- AI 是否直接吐结论而没有内核？
- 是否单脚本又解析又算又解释？
- 是否拿全历史拟合称"效果好"？
- 是否缺数据就编值？
- 报告里有没有源里没有的新数字？
- 随机部分是否固定了种子？
- LLM 是不是既算又解释？
```

---

## 给用户的一页纸速记
1. 别先写代码，先让 AI 当分析师（A 侦察）。
2. 让它把架构切成带验收 Gate 的 phase，并告诉你"一共几步"（B 拆解）。
3. 一次只做一个 phase，客观验收过关再进下一个（C 建造）。
4. 永远问那 5 问：怎么验证？缺数据怎么办？能溯源吗？过拟合/泄漏？哪些没验证？
5. 看到红旗（AI 直接给结论、编值、不可复现）立刻叫停。
