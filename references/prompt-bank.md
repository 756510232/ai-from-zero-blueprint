# 调用与自查手册（替代「让用户每次复制提问」）

> 旧版让用户反复问 AI「三层架构是什么 / 有哪些陷阱 / 需要什么数据」——这些已固化进
> `references/blueprint-template.md`，AI **主动套用**即可，用户无需再问。
> 本文件只保留两件必要的事：
> 1. 第零步如何触发 **brainstorming** 确认真实需求；
> 2. A / B / C 各阶段 AI 的**自查清单**（已默认套用模板，不再向用户索要）。

---

## 第零步：用 brainstorming 确认「你到底想要什么」

在动手前，**先加载 superpowers 的 `brainstorming` skill**，和用户澄清真实产品目标，再进 A 侦察。

**切记**：用户给的永远是「产品意图」（如"做一个张雪峰式高考志愿网页应用，输出给考生和家长"），
**不是**「复盘 / 复现某个系统」之类的元任务。AI 不得假设终点，终点由本阶段与用户共同确认。

**调用方式**：加载 `superpowers/skills/brainstorming/SKILL.md`，遵循其流程：
- 先理解当前项目状态（文件 / 文档 / 近期 commit）；
- **一次只问一个问题**（多选优先），逐步细化；
- 探索 **2-3 个方案**并给出推荐与理由；
- 设计分 **200-300 字小段**呈现，每段后确认是否对；
- 覆盖：架构 / 组件 / 数据流 / 错误处理 / 测试；
- 确认后写入 `docs/plans/YYYY-MM-DD-<topic>-design.md` 并 commit。

**brainstorming 必须澄清的「终点」输出**：
- 产品形态（网页 / CLI / API / 桌面端）
- 目标受众（谁用、解决他什么痛点）
- 成功标准（可衡量的验收口径）
- 硬约束（技术栈 / 合规 / 预算 / 数据可得性）
- 明确「不做什么」（YAGNI， ruthless 删减）

确认终点后，才进入 A 侦察——且 A 侦察基于已确认的终点，**主动**套模板，不再反问用户已知信息。

---

## A 侦察：AI 自查清单（主动产出，不追问用户已知信息）

- [ ] **概念表**：8-12 个领域术语 → 人话类比 + 在系统里的角色
- [ ] **三层落点**：套 `blueprint-template.md §1`，写明每层负责 / 不负责 + 内核零 LLM import
- [ ] **陷阱**：套 `blueprint-template.md §2` 的 6 类，给本项目实例 + 防法
- [ ] **数据**：套 `blueprint-template.md §3`，标形态 / 可得性 / 降级映射
- [ ] **PROVE**：每个关键说法给出「为什么可信」（出处 / 成熟做法 / 已知数据）
- [ ] 全程不向用户重复询问「三层是什么 / 陷阱有哪些」——这些是通用的，直接套

## B 拆解：AI 自查清单（套标准 phase，聚焦四要素）

- [ ] 套 `blueprint-template.md §4` 的 6 个 phase（P1–P6），填充本项目内容
- [ ] **先定档（S/M/L，见 §0）**：在拆解里写明本项目档位（A 侦察已请用户确认一行），后续强制动作按档套用
- [ ] **P1 强制（按 §0 分级）**：M/L 级写单一共享 schema 契约（输入+输出含降级态枚举）+ loader 断言表头===契约字段 + 读 CSV 前剥 `#`/空行；S 级可仅在代码顶部注释列字段
- [ ] **P2 强制（按 §0 分级）**：M/L 级先写锁方向/符号约定的单测再写业务逻辑；S 级改为注释写死方向约定 + 手算对照 3 样本（grep 零 LLM 抓不到逻辑错，单测/手算才抓得到）
- [ ] 每 phase 写清**四要素**：约束目标 / 完成标准 / 风险 / 检测方法
- [ ] 标注高风险 phase + 失败时的分支方案
- [ ] 自然得出**总步数**（基于固定模板的实例化，而非临时发明）
- [ ] 用户看这份拆解，重点看懂「每步边界与验收口径」，而非仅步骤名称

## C 建造：AI 自查清单（逐 phase 照四要素检测）

- [ ] **一次只做一个 phase**，对照其四要素验收，不过关不进下一 phase
- [ ] **P2 锁方向不过不进 P3（M/L 级硬门槛；S 级改为手算对照 3 样本不过不进 P3）**（量纲反转类 bug 在此拦下）
- [ ] 内核文件 `grep` 零 LLM / AI import（铁律自检）；并记 `grep` 仅防 LLM 混核，防不了逻辑错
- [ ] **每步写 `chat_log.md` 真实问题**（降级伪装 / 硬约束空过滤 / 跨口径误导等），它是反哺 Skill 的原材料
- [ ] 不偷偷做后续 phase；不一次性甩整套
- [ ] 收尾逼 AI 列**诚实声明**：已验证 / 只是推测 / 用了降级 / 复现所需（种子 / 数据版本 / 依赖）
- [ ] 出现反模式红旗（见 SKILL.md）立即回滚修正

## 给用户的一页纸速记（仅当他想亲自掌控节奏时用）

1. 先和我头脑风暴，把"到底要做个啥"聊清楚（产品形态 / 受众 / 成功标准 / 不做什么）。
2. 之后我会主动套三层骨架 + 陷阱 + 数据模板做侦察，不用你教我架构。
3. 拆解时我会给你"每步的约束 / 标准 / 风险 / 怎么检测"，你重点看这些边界。
4. 建造时一次一步，按上面的检测法验收，不过关就回滚。
5. 看到红旗（AI 直接给结论无内核 / 编值 / 不可复现）立刻叫停。
